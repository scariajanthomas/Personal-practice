# OOP
Class structure

```python
class User:
    def __init__(self, username, password):
        self.username = username
        self.password = password

    def display(self):
        # print username and password here
```

Key points:
- `__init__` runs automatically when you create an instance
- `self` refers to the current object
- `self.username = username` stores the value on the object


***
'self` refers to the **specific instance** calling the method.

When you do `user1.display()` — Python translates that to `User.display(user1)` internally. `self` is how the method knows which object's data to use.

Without `self`:
```python
def display():
    print(f"Username: {self.username}")  # self doesn't exist — crashes
```

With two instances:
```python
user1 = User("john", "secret123")
user2 = User("jane", "pass456")

user1.display()  # self = user1, prints john
user2.display()  # self = user2, prints jane
```
Same method, different data — `self` is what separates them.

# Inheritance and Composition

Inheritance — "is a" relationship
AdminUser is a User. It gets everything User has and adds more.
pythonclass AdminUser(User):  # AdminUser IS a User
    pass

Composition — "has a" relationship
User has a Session. Session is a separate class, stored inside User as an attribute.
pythonclass Session:
    pass

class User:
    def __init__(self):
        self.session = Session()  # User HAS a Session
